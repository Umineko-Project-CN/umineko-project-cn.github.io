## 技术问题与配置

### 图形显示异常
您的系统显卡驱动可能已过时。我们的引擎使用 OpenGL 而非 DirectX 开发。
因此，即使您系统上的其他游戏运行正常，过时的驱动也可能导致本游戏出现问题。请访问
[此处](https://www.umineko-project.org/en/known-issues/#video_issues)
获取详细的帮助信息。

### 遇到其他程序错误
首先，请确认该错误是否已知。如果是已知问题且我们未要求提供详细信息，您可以暂时搁置并等待修复。
否则，请按照错误报告指南，在论坛中创建相关主题进行反馈。

### 游戏控制方式
游戏界面设计较为直观：
- **主要操作：** 通常只需使用鼠标。
    - 左键点击：确认。
    - 右键点击：取消/返回。
- **快捷键：**
    - 全屏模式：按下键盘上的 <Kbd>F</Kbd> 键，或在菜单中设置（仅限 macOS）。
    - 跳过文本：使用 <Kbd>CTRL</Kbd> 键或 <Kbd>ALT</Kbd>+<Kbd>S</Kbd> 键。
    - 静音：使用 <Kbd>ALT</Kbd>+<Kbd>M</Kbd> 键可关闭所有声音。

### 配置游戏手柄
如果您的游戏手柄无法即插即用（我们仅原生支持 **DualShock 3/4、Steam Controller** 和 **FuSa 手柄**），您可能需要自行配置按键映射。

1.  **方法：** 在 `ons.cfg` 文件中添加 `pad-map=CONFIG_STR` 属性。`CONFIG_STR` 为以逗号分隔的按键代码，对应 PS3 手柄按键顺序如下：
    `TRIANGLE, CIRCLE, CROSS, SQUARE, L1, R1, DOWN, LEFT, UP, RIGHT, SELECT, START, HOME, L2, R2, L3, R3, EXTRA, DOWN, LEFT, UP, RIGHT`。
    使用 `-1` 表示无映射。
2.  **示例：** 
    ```ini
    gamepad_map=1,2,3,4,-1,-1,15,16,17,18,-1,-1,-1,-1,8,9,10,11
    ```
3.  **查找按键代码：** 若不清楚代码，可在 `ons.cfg` 中添加 `use-console` 和 `debug` 选项，然后启动游戏并使用手柄。按键代码将显示在控制台窗口中，方便您记录。

### 调整字体大小
如需调整游戏字体大小，您需要 **v3.3a** 或更新版本（Episode 3）。

- **方法：** 在 `ons.cfg` 文件中添加 `font-multiplier=CONFIG_STR` 属性，并设置相应参数。
- **示例：** 要将游戏文本放大 20%（1.2 倍），添加如下行：
    ```ini
    font-multiplier=b1:1.2,b2:1.2,b3:1.2,b4:1.2,b5:1.2,b6:1.2,b7:1.2,p0:6:1.2,p1:1:1.2,p1:2:1.2,p1:3:1.2,p1:4:1.2,p1:5:1.2,p1:7:1.2,p2:1:1.2,p2:2:1.2,p2:3:1.2,p2:4:1.2,p2:5:1.2,p2:7:1.2,p41:1:1.2,p41:2:1.2,p41:3:1.2,p41:4:1.2,p41:5:1.2,p41:7:1.2,p42:1:1.2,p42:2:1.2,p42:3:1.2,p42:4:1.2,p42:5:1.2,p42:7:1.2
    ```
- **自定义：** 将上述所有 `1.2` 替换为您想要的缩放系数。

### 不同游戏语言是否共享存档及设置？
**不共享。**

但您可以在 `ons.cfg` 中设置全局配置，或通过命令行参数传递。部分可用参数如下：

| 参数 | 取值范围 | 说明 |
| :--- | :--- | :--- |

| `env[force_game]` | chiru/rondo | 强制指定游戏模式 |
| `env[first_launch]` | passed/force | 控制首次启动流程 |
| `env[cg_unlock]` | 1-8 | 解锁指定章节的CG |
| `env[bgm_unlock]` | 1-8 | 解锁指定章节的BGM |
| `env[automode_hide_cursor]` | true/false | 自动模式下隐藏光标 |
| `env[bgm_title_language]` | English/Russian/Romaji/Japanese | BGM标题语言 |
| `env[skip_title]` | all/force | 跳过标题画面 |
| `env[video_resolution]` | 720/1080 | 视频分辨率 |
| `env[control_interface]` | kb/pad | 控制界面类型 |
| `env[object_caching]` | true/false | 对象缓存开关 |
| `env[bgm_volume]` | 0-100 | BGM音量 |
| `env[effect_volume]` | 0-100 | 音效音量 |
| `env[voice_volume]` | 0-100 | 语音音量 |
| `env[rumble_strength]` | 0-100 | 震动强度 |

| `env[skip_unread_dialogues]` | true/false | 跳过未读对话<br/>*需要 v3.3a 或更新版本* |

**查看所有可用参数：**
可通过在命令行执行 `--help` 参数查看完整的参数列表：
```bash
./onscripter-ru --help
```

**配置示例：**
```ini
# 在 ons.cfg 文件中设置
env[force_game]=chiru
env[effect_volume]=33
env[text_speed]=5
env[bgm_title_language]=Japanese

# 或通过命令行传递
./onscripter-ru --env[force_game] chiru --env[effect_volume] 33 --env[bgm_title_language] Japanese
```

### 更新后我的存档还能用吗？
- **Episode 5 及之后版本：** 更新后存档应该可以继续使用。
- **从更早章节升级时：** 仅奖杯和游戏进度会保留，具体存档将**无法加载**。这意味着您可以使用信息浏览器的行跳转功能继续游戏。

### 配置菜单中没有我的屏幕分辨率选项，怎么办？
您可以手动修改 `ons.cfg` 中的 `window-width` 参数来设置窗口宽度。

**请注意：**
- 游戏以 **16:9** 比例运行。请确保计算出的游戏高度（`window-width*9/16`）小于或等于您屏幕的物理分辨率高度。
- 对于高 DPI（HDPI）屏幕，您应提供物理分辨率。例如，如果您的屏幕缩放为 200%，那么您真实的屏幕分辨率是显示分辨率的一半。

### 如何将 Rondo 片头更改为 yu-yo works 制作的版本？
升级到 **4.2β** 或更高版本。然后在 `ons.cfg` 中添加 `env[legacy_op]=true` 参数，并重启游戏。

## 高级技术配置

### 引擎的其他可配置项
运行游戏时添加 `–help` 命令行参数，可以查看 `ons.cfg` 中各种有用的配置选项。

### 不喜欢文本框样式，如何更改？
您可以在 `graphics/system/wnd/` 文件夹中找到相关文件，并按喜好自由编辑。

**重要前提：** 您必须在 `ons.cfg` 文件中添加以下行：
```ini
env[verify]=none
```
添加此行意味着您将自行承担因修改游戏文件而可能导致的稳定性与兼容性风险。

### 不喜欢游戏字体，如何更改？
目前没有简单易用的方法来更改游戏字体。我们虽然提供了可自定义游戏中任何字体的接口，但这需要一定的技术知识和耐心。欢迎在论坛联系我们以获取详细信息。

### 我的 Linux 发行版不受支持，怎么办？
已知游戏在 **Wine** 环境下可以完美运行（我们上次测试时，Wine 1.7.21 工作正常）。建议使用 **1.6 分支或更新的最新版本**。

## iOS/Android 安装帮助

从 EP5 开始，我们提供了对 iOS（8.0+）和 Android（4.1+）设备的良好支持。

### 在 iOS 设备上安装
您可以使用 **Apple Developer Program 证书** 或 **Cydia Impactor** 应用程序来安装。为了传输游戏文件，推荐使用 **最新版本的 iTunes**。

> [Sideloadly](https://pangu8.com/sideloadly/) 可替代已过时的 Cydia Impactor 以便新版本 iOS 用户安装应用，请自行测试。

### 在 Android 设备上安装
您可以参考以下常见问题解答进行安装：

**游戏文件路径：**
游戏应存放在 SD 卡（模拟或外部）根目录的 `ONScripter-RU` 文件夹内。
*   **存档位置：** 所有存档都将存储在**内部（模拟）SD 卡**上，因为某些设备或固件无法写入外部存储卡。因此，如果游戏文件存放在外部 SD 卡上，则**不允许**在游戏文件夹中存在 `ons.cfg` 文件（请勿与 `default.cfg` 混淆）。
*   **路径问题：** 某些设备可能难以确定外部 SD 卡的路径，导致游戏启动失败。如果遇到此问题，请在内部（模拟）SD 卡的 `ONScripter-RU` 文件夹中创建 `default.cfg` 文件，并添加一行 `root=/完整/游戏/路径`（不带引号）。

**兼容性：**
游戏面向任何基于 Android 4.1 及以上系统、具有至少 **2 GB RAM** 和 **OpenGL ES 2.0** 支持（纹理尺寸限制至少为 4096 像素）的 armv7s、arm64 或 x86 架构设备。通常，我们推荐使用更新的操作系统版本。
*   **Android 4.3 及以上：** 在支持的设备型号上可以使用 OpenGL ES 3.0 渲染器。
*   **Android 4.4 及以上：** 可以使用完整的屏幕空间。
*   **Android 5.0 及以上：** 在支持的设备型号上可以利用 64 位 CPU 原生性能。

**修复视觉显示问题：**
某些设备需要在 `ons.cfg` 中添加 `render-self=no` 选项以修复画面异常。

**修复宽屏设备上的图像裁剪问题：**
您需要更新到 r3569 或更高版本，并在 `ons.cfg` 中指定用于将游戏画面左移的手动屏幕偏移量。例如，对于屏幕分辨率为 2400x1080 的 Redmi Note 9 Pro，应添加 `system-offset-x=180`。

**性能优化：**
目前存在多种可能影响性能的问题，因此建议您仔细配置以下选项：

1.  **选择渲染器：**
    默认情况下，ONScripter-RU 使用 **OpenGL ES 2.0** 渲染器。如果初始化失败，或者 `ons.cfg` 中存在 `renderer-blacklist=GLES2` 这行设置，则会回退到 **OpenGL ES 3.0** 渲染器。某些设备可能不支持 OpenGL ES 3.0，但启用它可能会带来稍好的性能。

2.  **启用硬件视频解码：**
    默认情况下，由于性能不佳，ONScripter-RU 在 Android 上禁用了硬件视频解码。但您可以通过在 `ons.cfg` 中添加 `hwdecoder=on` 来启用它，这可能有助于降低功耗。请注意，无论是否启用此选项，都无法保证在任何移动设备上都能流畅播放 1080p 视频。

3.  **后备纹理加载：**
    在某些设备上使用 OpenGL ES 2.0 渲染器时，您可以通过启用额外的 RAM 缓存来改善特定纹理（例如配置菜单）的加载性能。为此，请在 `ons.cfg` 中添加 `texture-upload=ramcopy`。请注意，这可能导致内存不足的低端设备崩溃。使用 OpenGL ES 3.0 渲染器时，此设置将被忽略。

4.  **帧率限制：**
    您可以通过将游戏帧率限制在低于 60 帧/秒的值来改善性能并显著降低功耗。请在 `ons.cfg` 中添加 `force-fps=30`，其中 30 是您期望的帧率值。您还可以额外添加 `force-vsync` 来启用旧式垂直同步，但这不太可能带来明显益处。不设置帧率限制可能导致在某些硬件上的部分场景中出现黑条等渲染异常。

5.  **纹理尺寸限制：**
    通过将支持的纹理尺寸限制设置为低于设备本身支持的值，您可能会在某些场景中获得稍好的性能。请在 `ons.cfg` 中添加 `texlimit=4096`，其中 4096 是以像素为单位的 2 的幂次方值。

**获取日志文件：**
您需要为设备安装相应的 ADB 驱动程序，然后使用命令行工具获取必要输出。例如，在类 UNIX 系统上，您可以使用：
```bash
adb logcat | grep --line-buffered -Ei 'sdl|onscripter' > /path/to/my/log.txt
```
